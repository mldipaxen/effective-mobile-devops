## Технологии

-   Docker и Docker Compose\
-   Python 3.12-slim (официальный минимальный образ)\
-   Nginx 1.25-alpine (официальный минимальный образ)\
-   Изолированная Docker-сеть (bridge driver)\
-   Healthchecks для контроля готовности сервисов

------------------------------------------------------------------------

## Дополнительные меры надёжности и безопасности

Помимо требований ТЗ, реализованы следующие меры:

-   Запуск backend от непривилегированного пользователя `appuser`\
-   Файловая система контейнеров переведена в режим `read_only`,
    временные директории вынесены в `tmpfs`\
-   Опция `no-new-privileges:true` запрещает повышение прав внутри
    контейнера\
-   Удаление лишних Linux-капабилити (`cap_drop: [ALL]`) для backend\
-   Скрытие версии Nginx в HTTP-заголовках (`server_tokens off`)

------------------------------------------------------------------------

## Дополнительная функциональность

-   Реализован graceful shutdown в Python-приложении (обработка
    `SIGTERM`)\
-   Healthchecks в `Dockerfile` и `docker-compose.yml` (интервал,
    таймаут, период запуска)\
-   `depends_on: condition: service_healthy` --- запуск nginx только
    после готовности backend\
-   Ограничение размера логов:
    -   10 МБ на файл\
    -   максимум 3 файла\
-   Явные лимиты CPU и памяти через `deploy.resources.limits`\
-   Конфигурируемые параметры (версии образов, порты, лимиты) вынесены в
    `.env`

## CI/CD

При каждом `push` в ветку `main` или создании `pull_request`
автоматически запускается пайплайн.

### Проверки:

-   Lint Dockerfile --- поиск антипаттернов через `hadolint`\
-   Сборка и запуск --- `docker compose up -d --build`\
-   Functional test --- `curl` проверяет точный ответ\
-   Isolation test --- проверка недоступности `:8080` с хоста\
-   Логи и очистка --- отладка и освобождение ресурсов

## Примечание

Для корректной работы CI:

-   В `docker-compose.yml` добавлены дефолтные значения переменных\
-   Конфигурация nginx проверяется только на наличие

Причина:

`docker run nginx -t` не работает без поднятой compose-сети, так как имя
`backend` резолвится только внутри Docker Compose сети благодаря
встроенному DNS-серверу Docker (`127.0.0.11`).

Обычный `docker run` использует DNS хоста или дефолтную bridge-сеть, где
имени `backend` не существует.
